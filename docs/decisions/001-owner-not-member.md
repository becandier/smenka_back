# 001 — Владелец организации не является участником

Статус: принято
Фаза: 3
Влияет на: фаза 4 (статистика по организации, геопроверка)

## Контекст

При проектировании модели организации возник вопрос: должен ли создатель (owner) автоматически добавляться в таблицу `organization_members`?

## Решение

Owner хранится только в поле `Organization.owner_id` и **не** добавляется в `organization_members`. Причина: владелец — это администратор, которому не нужно отслеживать рабочее время. Он управляет организацией, но не является сотрудником.

Это означает:
- Owner имеет полные права (CRUD орг, ротация ��нвайта, управление участниками) через проверку `owner_id`
- Owner не может присоединиться к своей организации по инвайт-коду (`OWNER_CANNOT_JOIN`)
- При запросе списка участников owner не отображается в списке
- Для проверки доступа к организации используется `_check_org_access`: сначала проверяется `owner_id`, затем наличие в `organization_members`

## Последствия

- **Фаза 4**: при реализации `GET /organizations/{id}/shifts` (смены сотрудников для админа) — owner сможет видеть смены, но своих смен в контексте организации у него нет
- **Фаза 4**: `GET /organizations/{id}/stats` — статистика только по участникам из `organization_members`, без owner
- Если в будущем потребуется, чтобы owner тоже трекал время — его нужно будет добавить как member с ролью `admin`
